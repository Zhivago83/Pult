# Бриф «Пульта руководителя»

Этот документ — **источник истины** по продукту. Каждый слой делается строго
по брифу из этого файла; всё, чего здесь нет, в приложение не добавляется.
Новые слои дописываются в конец по мере поступления от владельца продукта.

Формулировки сохранены авторские (владельца продукта), без пересказа.

---

## Продукт

«Пульт руководителя» — личный планировщик для одного руководителя отдела:
в одном месте все его задачи, ожидания от команды и проекты.
Кроссплатформенность и синхронизация — цель на будущее; сейчас — веб-приложение,
которое ставится на телефон/айпад как иконка и работает офлайн.

**Стек:**
- React + TypeScript + Vite, устанавливаемый PWA (офлайн).
- Локальное хранилище за интерфейсом Store — реализация на IndexedDB (библиотека idb).
- С первого шага — журнал операций (op-log): каждое изменение записывается и
  может быть отменено (Undo). Undo вместо диалогов «вы уверены?».

**Дизайн (аскетичный, не шаблонный):**
- Почти монохром: графит на бумаге. Цвет только у «горящего» — красный.
  Остальное графитовое.
- Данные (сроки, счётчики) — шрифт JetBrains Mono; текст — Golos Text.
- Структура на воздухе и тонких линиях, без рамок-карточек и теней.
  Слева у пункта кружок-кольцо, у горящего — красное.

## Общие правила (обязательны для каждого слоя)

- Любой ввод текста — ТОЛЬКО через поля внутри интерфейса (шторки/панели).
  НИКОГДА не использовать системные окна браузера prompt/alert/confirm —
  на телефоне они не работают.
- Каждое изменение — через журнал операций (op-log) и с возможностью Undo
  (плашка «Отменить»). Никаких диалогов «вы уверены?».
- Сохранять аскетичный дизайн (см. выше).
- Данные хранятся локально (IndexedDB) и работают офлайн.
- Не ломать уже работающее из предыдущих слоёв.

## Первая версия — сделано ✅

1. Захват (кнопка +): создать пункт — «моё» или «жду от кого-то»; сразу
   появляется в Сводке.
2. Экран «Сводка»: сверху «термометр дня» (тонкая полоса + строка
   «N горят · M ждут · K моих»); ниже секции по тревожности:
   «Просрочено и горит» → «Пора пнуть» → «Моё» → «Ожидания».
   Пустой экран — спокойный, без цвета.
3. Закрытие пункта тапом по кружку слева с «периодом благодати»:
   ~6 секунд пункт зачёркнут, потом исчезает.
4. Undo: на любое действие (создание, закрытие, удаление) внизу плашка «Отменить».
5. Мягкое удаление в «Корзину» с восстановлением.
6. Данные сохраняются и работают офлайн.
7. Тема: светлая «Бумага» и тёмная «Консоль», переключатель.

## Слой 1: полный экран пункта (карточка дела) — сделано ✅

1. Тап по пункту в Сводке открывает его карточку (на телефоне — шторка снизу;
   на широком экране — панель справа).
2. В карточке:
   - заголовок пункта, правится тапом;
   - редактируемые «пилюли»: владелец (кто), срок, проект — тап открывает
     выбор и меняет значение;
   - видно вид пункта (моё / жду).
3. История касаний — лента событий пункта по времени: системные записи
   («создано», «срок изменён» и т.п.) и комментарии, вперемешку.
4. Кнопка «Комментарий» — добавляет запись в историю.
5. Кнопка действия: для «жду» — «Отметить напомнил»; для «моё» — «Выполнено»
   (с тем же периодом благодати, что уже есть).

## Слой 2: экран «Жду» с разбивкой по людям и карточка человека — сделано ✅

1. На экране «Жду» — переключатель «Список / По людям».
2. Режим «По людям»: две группы —
   - «Команда» (люди, которым я делегирую),
   - «Исполнители» (внешние, от кого я просто жду).
   В строке человека: имя, сколько от него жду, сколько просрочено, и
   «надёжность» словами (например: обычно вовремя / тянет с ответом /
   срывал срок). «Срывал срок» — единственная красная пометка, остальное графитом.
3. Тап по человеку открывает его карточку:
   - имя и роль (команда/исполнитель), надёжность;
   - список всех ожиданий от него — тап по ожиданию открывает карточку пункта
     (из прошлого слоя).
4. Надёжность считается из истории: если человек переносил свои сроки —
   «тянет/срывал»; если обычно закрывает вовремя — «обычно вовремя».
   Расчёт простой.

## Слой 3: Проекты — сделано ✅

1. Новая вкладка «Проекты» рядом со «Сводка» и «Жду».
2. Список проектов; у каждого — название и тонкая полоса прогресса
   (сколько пунктов closed из всех).
3. Тап по проекту открывает раскрытый проект: его пункты, сгруппированные —
   «просрочено», «в работе», и свёрнутое «сделано». Тап по пункту открывает
   уже существующую карточку пункта.
4. У пункта поле «проект» (пилюля в карточке) позволяет выбрать проект из
   списка существующих или создать новый — только через поля внутри интерфейса.
5. На «Сводке» внизу — короткая строка «Проекты в движении» со списком
   активных (реализована как «В движении: …»).

## Примечание об истории

(проекты в другой трактовке, напоминания, умный захват, повторы, заметка дня,

вернулась к движению строго по брифу. Код надстроек сохранён в истории git
(PR #6–#17) и при желании любой из них можно вернуть отдельным решением.
