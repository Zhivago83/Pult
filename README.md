# Пульт руководителя

Личный планировщик руководителя отдела: в одном месте все задачи, ожидания от
команды и проекты. Первая версия — устанавливаемое веб-приложение (PWA),
которое работает офлайн и ставится на телефон/айпад как иконка.

## Что уже умеет

- **Захват (+)** — быстро создать пункт: «моё» или «жду от кого-то».
- **Сводка** — «термометр дня» и секции по тревожности: «Просрочено и горит» →
  «Пора пнуть» → «Моё» → «Ожидания». Пустой экран — спокойный, без цвета.
- **Закрытие тапом** по кружку слева с «периодом благодати» (~6 c зачёркнуто,
  потом исчезает).
- **Undo** на любое действие — плашка «Отменить» вместо диалогов «вы уверены?».
- **Корзина** — мягкое удаление с восстановлением.
- **Офлайн** — данные в IndexedDB, приложение открывается без сети.
- **Две темы** — светлая «Бумага» и тёмная «Консоль».

## Как устроено (слоями)

```
src/
  types.ts            типы данных: Item (пункт), Op (запись журнала)
  store/store.ts      хранилище: интерфейс Store + реализация на IndexedDB (idb)
  core/               ядро: op-log, период благодати, производная логика Сводки
    constants.ts        настройки (период благодати, пороги «горит»/«пнуть»)
    derive.ts           сборка секций и термометра из списка пунктов
    seed.ts             демо-пункты для первого запуска
  state/engine.tsx    движок: действия + журнал операций + Undo
  ui/                 экраны: Сводка, Захват, Тост-Undo, Корзина, темы
```

Каждое изменение проходит через журнал операций (op-log): сохраняется снимок
«до» и «после», поэтому любое действие можно отменить.

## Запуск на своём компьютере

```bash
npm install
npm run dev      # откроется на http://localhost:5173
```

## Сборка

```bash
npm run build    # результат в папке dist/
npm run preview  # посмотреть собранную версию
```

## Публикация

При пуше в рабочую ветку GitHub Actions собирает проект и публикует его на
GitHub Pages (см. `.github/workflows/deploy.yml`). Адрес:
`https://<логин>.github.io/Pult/`.
